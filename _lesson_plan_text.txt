คำนำ
แผนการจัดการเรียนรู้วิชา การพัฒนาซอฟต์แวร์รูปแบบเดฟออฟส์ (DevOps) รหัสวิชา 30901-2008 ฉบับปรับปรุงนี้ จัดทำขึ้นบนพื้นฐานของหลักสูตรประกาศนียบัตรวิชาชีพชั้นสูง พุทธศักราช 2567 โดยปรับเปลี่ยนแนวทางการจัดกิจกรรมจาก "หน่วยเรื่อง ๆ จบไป" ไปสู่แนวทาง Spiral Curriculum ที่ผู้เรียนพัฒนา Application ชิ้นเดียวกันตลอดภาคเรียน แล้วค่อย ๆ ห่อหุ้มด้วย DevOps Practice ทีละชั้นในแต่ละสัปดาห์ ทำให้ผู้เรียนได้ "ส่งชิ้นงาน" ทุกสัปดาห์
แผนนี้ออกแบบมาให้สอดคล้องกับธรรมชาติของผู้เรียนระดับ ปวส. สาขาเทคโนโลยีสารสนเทศ ที่ต้องการความถี่ในการฝึกที่บ่อยครั้งและเห็นผลลัพธ์ที่จับต้องได้ นอกจากนี้ยังประยุกต์ใช้ทรัพยากร Proxmox ของวิทยาลัยอย่างเต็มที่ — ทั้ง Private Registry, VM Pool สำหรับ Staging/Production, k3s Cluster และ Monitoring Stack — เพื่อจำลองสภาพแวดล้อมการทำงานจริงในองค์กรไทยที่ส่วนใหญ่ยังเป็น On-premise
แผนการจัดการเรียนรู้ฉบับนี้ประกอบด้วยหน่วยการเรียนรู้ทั้งหมด 17 หน่วย (ตามจำนวนสัปดาห์เรียน) ครั้งละ 5 ชั่วโมง รวม 85 ชั่วโมง โดยมีสอบกลางภาคเป็นการสอบเชิงปฏิบัติ (Practical Exam) ในสัปดาห์ที่ 8 และการนำเสนอ Capstone Project พร้อมสอบปลายภาคในสัปดาห์ที่ 17 เนื้อหาครอบคลุมตั้งแต่พื้นฐาน DevOps, Git, Fullstack Application, Containerization, CI/CD, Kubernetes, Monitoring, Infrastructure as Code และ DevSecOps

หลักการจัดการเรียนรู้ (Spiral Curriculum)
ผู้เรียนพัฒนา Application หนึ่งตัวร่วมกันเป็นทีม 3-4 คน ตลอดภาคเรียน โดยในแต่ละสัปดาห์จะเพิ่ม DevOps Practice หนึ่งชั้นลงบน Application เดิม (Git → Fullstack → Container → Registry → GitFlow → CI → Quality Gate → CD → K8s → HPA → Strategies → Monitor → Logging → IaC → Security → Capstone) ผู้เรียนจึงมี deliverable ส่งทุกสัปดาห์และเห็นพัฒนาการของระบบตนเองอย่างต่อเนื่อง

หน่วยการเรียนรู้รายสัปดาห์
ในส่วนต่อไปนี้เป็นแผนการจัดการเรียนรู้รายสัปดาห์ทั้ง 17 สัปดาห์ ตามแนวทาง Spiral Curriculum โดยทุกสัปดาห์มี deliverable ที่ผู้เรียนต้องส่ง และ Application ของทีมจะค่อย ๆ ถูกห่อหุ้มด้วย DevOps Practice เพิ่มขึ้นทีละชั้น

หน่วยที่ 1

รหัสวิชา 30901-2008  ชื่อวิชา การพัฒนาซอฟต์แวร์รูปแบบเดฟออฟส์ (DevOps)
สอนครั้งที่ 1

ชื่อหน่วยการเรียนรู้  แนวคิดและหลักการ DevOps (Foundation)
ทฤษฎี 1 ชม. ปฏิบัติ 4 ชม.

1. ผลลัพธ์การเรียนรู้ระดับหน่วยการเรียน
ผู้เรียนสามารถอธิบายแนวคิด หลักการ วัฒนธรรม DevOps Lifecycle และใช้ Git/GitHub เพื่อจัดการ Source Code ของตนเองได้
2. อ้างอิงมาตรฐาน/เชื่อมโยงกลุ่มอาชีพ
2.1 มาตรฐานอาชีพ undefined
2.2 บูรณาการกลุ่มอาชีพ เทคโนโลยีสารสนเทศและการสื่อสาร
3. สมรรถนะประจำหน่วย
3.1  ประมวลความรู้เกี่ยวกับแนวคิด CALMS และ DevOps Lifecycle
3.2  ใช้ Git CLI พื้นฐานในการจัดการ Repository ส่วนตัวบน GitHub

4. จุดประสงค์เชิงพฤติกรรม
(พุทธิพิสัย: K2,K3 | ทักษะพิสัย: S2 | จิตพิสัย: A3 | ประยุกต์ใช้: Ap2)
4.1  [K2,K3] อธิบายหลักการ CALMS และ DevOps Lifecycle พร้อมยกตัวอย่างองค์กรที่ประสบความสำเร็จได้
4.2  [S2] ใช้คำสั่ง git init, add, commit, push, pull, branch ได้อย่างถูกต้อง
4.3  [A3] เห็นคุณค่าของการทำงานร่วมกันระหว่าง Dev และ Ops และมีวินัยในการเขียน Commit Message
4.4  [Ap2] สร้าง GitHub Profile Repository พร้อม README และ commit history ที่อ่านง่ายได้

5. สาระการเรียนรู้
1.1 ความหมาย ที่มา และวิวัฒนาการของ DevOps
1.2 หลักการ CALMS (Culture, Automation, Lean, Measurement, Sharing)
1.3 DevOps Lifecycle และ Infinity Loop
1.4 เปรียบเทียบ Waterfall, Agile, Scrum และ DevOps
1.5 Git Architecture (Working Tree, Index, Local Repo, Remote)
1.6 Git Commands พื้นฐาน: init, clone, add, commit, push, pull, branch
1.7 GitHub Workflow: Repository, README, .gitignore, Commit Convention

6. กิจกรรมการเรียนรู้ (Active Learning)
ครั้งที่ 1 — DevOps Concept + Git Onboarding (300 นาที)
• Set the Stage (20 นาที): ครูฉาย Meme "It works on my machine" → ผู้เรียนแชร์ประสบการณ์ปัญหาส่งโค้ดกันในทีม
• Concept Input (40 นาที): บรรยาย CALMS + DevOps Lifecycle ผ่าน Infographic — เน้นว่าตลอดเทอมจะค่อย ๆ เพิ่ม practice ทีละชั้นบน "app เดียว"
• Guided Lab — Git First Touch (90 นาที): ผู้เรียนสมัคร GitHub → ตั้งค่า SSH Key → ฝึก init/add/commit/push บนเครื่องตนเอง
• Deliverable Sprint (120 นาที): สร้าง GitHub Profile Repository (username/username) พร้อม README แนะนำตัว + มี commit ≥ 5 ครั้งที่มี message สื่อความหมาย
• Exit Ticket + Demo (30 นาที): ผู้เรียนทุกคนแชร์ลิงก์ profile ของตน + เขียน 3 สิ่งที่เรียนรู้/สงสัย

7. สื่อและแหล่งการเรียนรู้
1. คอมพิวเตอร์พร้อม Git, VS Code, GitHub Account
2. สไลด์ Infographic CALMS + DevOps Lifecycle
3. Worksheet Git Step-by-step
4. Profile Repository Rubric

8. หลักฐานการเรียนรู้
8.1 หลักฐานความรู้:  Exit Ticket DevOps Concept + Git Cheat Sheet ที่ผู้เรียนเขียนเอง
