โครงสร้าง Frontend + vite.config.js + main.js
Setup

ใน Step นี้จะ สร้าง Vue 3 + Vite project ด้วยตัวเอง ตั้งแต่ต้น — ไม่ต้องเขียน package.json เอง ใช้ Vite scaffold ให้เลย

1. Scaffold โปรเจกต์ด้วย Vite:

terminal — อยู่ใน week3-containerize/
npm create vite@latest frontend -- --template vue
💡
คำสั่งนี้ทำอะไร? npm create vite@latest เรียกใช้ Vite project generator
frontend คือชื่อโฟลเดอร์ที่จะสร้าง
--template vue บอก Vite ว่าให้ scaffold Vue 3 + JavaScript template
Vite จะสร้าง package.json, vite.config.js, src/main.js และไฟล์อื่น ๆ ให้ครบ

โครงสร้างที่ Vite สร้างให้:

โครงสร้างไฟล์หลังรัน scaffold
frontend/
├── index.html            ← entry HTML, มี <div id="app">
├── package.json          ← dependencies (vue, vite, @vitejs/plugin-vue)

├── public/

2. ติดตั้ง dependencies:

terminal
cd frontend
npm install

package.json ที่ Vite สร้างให้มีอะไรบ้าง:

vueVue 3 framework — library หลักสำหรับ UI, reactive state, component system
viteBuild tool + Dev server — npm run dev รัน dev server, npm run build สร้าง production build
@vitejs/plugin-vueVue plugin — ทำให้ Vite อ่าน .vue files ได้ ถ้าไม่มีตัวนี้ Vite จะไม่รู้จัก syntax ของ Single-File Component
📦
dependencies vs devDependencies ต่างกันอย่างไร? dependencies (vue) — ต้องการตอน runtime ด้วย, ถูก bundle เข้า production build
devDependencies (vite, plugin-vue) — ใช้แค่ตอน develop/build, ไม่ถูก bundle เข้า production
✅
ตรวจสอบหลัง npm install
  • มีโฟลเดอร์ frontend/node_modules/ ถูกสร้างขึ้น
  • มีไฟล์ frontend/package-lock.json (lock versions)
  • ใน node_modules/ จะเห็นโฟลเดอร์ vue/, vite/, @vitejs/

import { defineConfig } from 'vite'

เขียน <script setup> — ส่วน Import + State:

เขียนใน: frontend/src/App.vue → <script setup>
Vue 3 Vanilla JS เปรียบเทียบ — การเก็บ state
// ❌ Vanilla JS — ต้องเก็บใน variable ธรรมดา DOM ไม่อัพเดตอัตโนมัติ let products = [] products = await res.json() // ต้อง rebuild DOM เอง... // ✓ Vue 3 — ref() สร้าง reactive state, DOM อัพเดตอัตโนมัติ const products = ref([]) // array ว่าง products.value = await res.json() // เปลี่ยนค่าผ่าน .value // Vue detect การเปลี่ยนแปลง → re-render เฉพาะส่วนที่ใช้ products
App.vue — <script setup> ส่วน State
import { ref, computed, onMounted } from 'vue'

// ── State ────────────────────────────────────────────────────────
const products      = ref([])       // รายการสินค้าทั้งหมดจาก API
const loading       = ref(true)     // แสดง loading spinner
const search        = ref('')       // ข้อความค้นหา
const catFilter     = ref('')       // category ที่กำลัง filter
const showModal     = ref(false)    // แสดง/ซ่อน modal เพิ่ม/แก้ไข
const editingId     = ref(null)     // null = กำลัง add, มีค่า = กำลัง edit
const confirmDelete = ref(null)     // เก็บ product ที่จะลบ (ใช้ confirm dialog)

// ข้อมูลในฟอร์ม modal
const form = ref({ name: '', category: '', price: '', stock: 0, description: '' })

let debounceTimer = null            // สำหรับ debounce search input

อธิบาย State แต่ละตัว:

productsเก็บ array ของสินค้าจาก API — เมื่อเปลี่ยน Vue จะ re-render product grid อัตโนมัติ
loadingควบคุมการแสดง spinner "กำลังโหลด..." — true ตอนเริ่ม, false หลัง fetch เสร็จ
searchผูกกับ input field ด้วย v-model — เมื่อ user พิมพ์ ค่านี้จะอัพเดตทันที
catFilterผูกกับ select dropdown — ใช้กรองสินค้าตาม category
showModalควบคุมการแสดง modal form — v-if="showModal" ใน template
editingIdnull = mode Add, มีค่า id = mode Edit — ใช้ตัดสินใจว่าจะ POST หรือ PUT
confirmDeleteเก็บ product object ที่กำลังจะลบ — ใช้แสดง confirm dialog
formเก็บข้อมูลที่กรอกในฟอร์ม modal ทุก field ผูกด้วย v-model
📌
การอ่าน/เขียนค่า ref() ต้องใช้ .value เสมอ ใน script: products.value = [...] / loading.value = false
ใน template: Vue ทำให้อัตโนมัติ ไม่ต้อง .value — {{ products.length }} ก็พอ

App.vue — Script Setup: Computed
Vue 3

computed() คือค่าที่คำนวณจาก state — Vue จะ recalculate เฉพาะเมื่อ dependencies เปลี่ยน และ cache ผลไว้

เขียนต่อใน: frontend/src/App.vue → <script setup> ต่อจาก State
App.vue — Computed properties
// ── Computed ─────────────────────────────────────────────────────

// กรองสินค้าตาม search + catFilter พร้อมกัน
const filtered = computed(() => {
  const s = search.value.toLowerCase()
  return products.value.filter(p => {
    // ตรงกับ search text? (เช็คทั้ง name และ description)
    const matchS = !s ||
      p.name.toLowerCase().includes(s) ||
      (p.description || '').toLowerCase().includes(s)
    // ตรงกับ category filter?
    const matchC = !catFilter.value || p.category === catFilter.value
    return matchS && matchC
  })
})

// สร้าง list ของ categories จาก products ที่มีอยู่ (unique + sort)
const categories = computed(() =>
  [...new Set(products.value.map(p => p.category))].sort()
)

// คำนวณ 4 ตัวเลข dashboard stats
const stats = computed(() => ({
  total:      products.value.length,
  lowStock:   products.value.filter(p => p.stock < 10).length,
  totalItems: products.value.reduce((s, p) => s + p.stock, 0),
  totalValue: products.value.reduce((s, p) => s + parseFloat(p.price) * p.stock, 0)
}))

อธิบายแต่ละ computed:

filteredอัพเดตทันทีเมื่อ search หรือ catFilter เปลี่ยน — template ใช้ v-for="p in filtered" ไม่ใช่ products โดยตรง
categoriesดึง category ที่ unique จาก products ทั้งหมด — ใช้ Set เพื่อตัดซ้ำ, sort() เรียงตัวอักษร
stats.totalจำนวนสินค้าทั้งหมด
stats.lowStockสินค้าที่ stock < 10 ชิ้น — ใช้แสดง alert banner
stats.totalItemsรวมจำนวน stock ทั้งหมดทุกสินค้า (reduce)
stats.totalValueมูลค่ารวม = ราคา × stock ทุกสินค้า
🧮
computed() vs methods() — ทำไมถึงใช้ computed? computed() มี cache — ถ้า products ไม่เปลี่ยน Vue ใช้ค่าเดิม ไม่คำนวณใหม่
methods() รันทุกครั้งที่ template re-render — สิ้นเปลืองกว่าสำหรับ stats ที่ต้องแสดงตลอด
📋
categories ไม่ได้มาจาก /api/categories App.vue คำนวณ categories จาก products ที่ดึงมาแล้ว — ไม่ต้องเรียก API เพิ่ม (backend มี /api/categories route ไว้ แต่ frontend ไม่ได้ใช้งานจริง)

App.vue — Script Setup: Methods + Template
Vue 3

ส่วน methods ทำงานกับ API (fetch, add, edit, delete) — เขียนต่อจาก Computed แล้ว Step นี้จะปิด <script setup> และเขียน <template> + <style scoped> ให้ครบ

เขียนต่อใน: frontend/src/App.vue → <script setup> ต่อจาก Computed
App.vue — Methods ทั้งหมด
// ── Methods ───────────────────────────────────────────────────────

// debounce: รอ 280ms หลัง user หยุดพิมพ์แล้วค่อย fetch
// (ป้องกัน API ถูกเรียกทุก keystroke)
function debounceFetch() {
  clearTimeout(debounceTimer)
  debounceTimer = setTimeout(fetchProducts, 280)
}

// ดึงสินค้าจาก API (ส่ง search + catFilter เป็น query params)
async function fetchProducts() {
  loading.value = true
  const q = new URLSearchParams()
  if (search.value)    q.set('search',   search.value)
  if (catFilter.value) q.set('category', catFilter.value)
  try {
    const res = await fetch(`/api/products?${q}`)
    products.value = await res.json()
  } catch {
    products.value = []
  } finally {
    loading.value = false
  }
}

// เปิด modal แบบ Add (clear form)
function openAdd() {
  editingId.value = null
  form.value = { name: '', category: '', price: '', stock: 0, description: '' }
  showModal.value = true
}

// เปิด modal แบบ Edit (copy ข้อมูลจาก product เข้า form)
function openEdit(p) {
  editingId.value = p.id
  form.value = {
    name: p.name, category: p.category,
    price: p.price, stock: p.stock,
    description: p.description || ''
  }
  showModal.value = true
}

// บันทึก (ถ้า editingId มีค่า = PUT, ถ้าเป็น null = POST)
async function saveProduct() {
  const body = {
    name:        form.value.name,
    category:    form.value.category,
    price:       parseFloat(form.value.price),
    stock:       parseInt(form.value.stock),
    description: form.value.description
  }
  const url    = editingId.value ? `/api/products/${editingId.value}` : '/api/products'
  const method = editingId.value ? 'PUT' : 'POST'
  await fetch(url, { method, headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(body) })
  showModal.value = false
  fetchProducts()   // reload สินค้าใหม่
}

// ลบสินค้า
async function deleteProduct(id) {
  await fetch(`/api/products/${id}`, { method: 'DELETE' })
  confirmDelete.value = null
  fetchProducts()
}

// คืน CSS class ตาม stock level (ใช้กับ stock bar และตัวเลข)
function stockClass(s) {
  if (s <= 0) return 'out'    // หมด
  if (s < 10) return 'low'    // ใกล้หมด
  if (s < 30) return 'mid'    // ปานกลาง
  return 'high'               // เพียงพอ
}

// คืน CSS class ตาม category (ใช้กับ badge สี)
function catClass(cat) {
  const m = {
    Electronics: 'c-elec', Clothing: 'c-cloth',
    Footwear:    'c-foot',  Food:     'c-food',
    Sports:      'c-sport', Books:    'c-book',
    Furniture:   'c-furn',  Bags:     'c-bag',
    Accessories: 'c-acc',   Tools:    'c-tool'
  }
  return m[cat] || 'c-other'
}

// โหลดสินค้าทันทีเมื่อ component mount ครั้งแรก
onMounted(fetchProducts)

Vue Directives ที่ใช้ใน Template:

v-for="p in filtered"วน loop ผ่าน filtered computed (ไม่ใช่ products โดยตรง) — search/filter มีผลทันที
:key="p.id"Vue ใช้ key track ว่า element ไหนคือ element ไหน — ลดการ re-render ที่ไม่จำเป็น
v-model="search"two-way binding — input เปลี่ยน search เปลี่ยน, search เปลี่ยน input แสดงค่าใหม่
v-if="showModal"สร้าง/ทำลาย DOM element ตามเงื่อนไข — ต่างจาก v-show ที่แค่ toggle display:none
@click.self="...".self modifier — event trigger เฉพาะ element นั้นเอง ไม่ใช่ child — คลิก overlay ปิด modal แต่คลิกใน modal ไม่ปิด
:class="stockClass(p.stock)"dynamic class — ผูก CSS class ตามผลลัพธ์ function
:style="{ width: ... + '%' }"dynamic style — stock bar width คำนวณจาก stock / 80 * 100
