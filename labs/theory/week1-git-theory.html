📚1. Version Control System คืออะไร?

Version Control System (VCS) คือระบบที่บันทึกการเปลี่ยนแปลงของไฟล์ตามเวลา ทำให้เราสามารถย้อนกลับไปดู หรือกู้คืน version ก่อนหน้าได้ รวมถึงทำงานร่วมกับคนอื่นในเวลาเดียวกันได้โดยไม่ชนกัน

🗓️ ไม่มี VCS (ยุคเก่า)
บันทึกไฟล์เป็น backup เอง เช่น project_v1.zip, project_final.zip, project_final2.zip — ไม่รู้ว่า version ไหนคืออะไร ทำงานทับกันบ่อย
📁 Local VCS
บันทึก changes บนเครื่องตัวเอง เช่น RCS — ยังทำงานร่วมกับคนอื่นไม่ได้
🌐 Centralized VCS
มี server กลาง เช่น SVN, CVS — ทุกคน checkout จาก server เดียวกัน แต่ถ้า server ล่มก็ทำงานต่อไม่ได้
🔀 Distributed VCS (Git)
ทุกคนมี full history บนเครื่องตัวเอง ทำงาน offline ได้ — Git คือตัวอย่างที่นิยมที่สุดในโลก
🏛️2. Git Architecture — 4 พื้นที่หลัก

Git แบ่งการทำงานออกเป็น 4 พื้นที่ ไฟล์เดินทางผ่านพื้นที่เหล่านี้ตามลำดับ

┌─────────────────────────────────────────────────────────────────────┐ │ เครื่องของเรา (Local) │ │ │ │ ┌─────────────┐ git add ┌──────────────┐ git commit ┌────────┐ │ │ │ Working Tree │ ──────────► │ Staging Area │ ────────────► │ Local │ │ │ │ (ไฟล์จริง) │ │ (Index) │ │ Repo │ │ │ └─────────────┘ └──────────────┘ └────────┘ │ │ git checkout/restore ◄──────────── git log ─────── │ └──────────────────────────────┬──────────────────────────────────────┘ │ git push / git pull ▼ ┌──────────────────────┐ │ Remote Repository │ │ (GitHub/GitLab) │ └──────────────────────┘
Working Tree
ไฟล์จริงๆ ที่เราแก้ไขบนเครื่อง — สถานะ: Untracked (ใหม่) หรือ Modified (แก้แล้ว)
Staging Area (Index)
พื้นที่กลางก่อน commit — เราเลือกได้ว่าจะ commit ไฟล์ไหนบ้างใน commit นี้
Local Repository
ประวัติ commits ทั้งหมดบนเครื่องเรา อยู่ใน folder .git/
Remote Repository
Server กลางบน GitHub/GitLab ที่ทีมทุกคน push/pull มาแชร์กัน
🔑3. วิธีที่ Git เก็บข้อมูล — SHA-1 Hash

Git ไม่ได้เก็บ "diff" เหมือน SVN — Git เก็บ snapshot ของทุกไฟล์ในแต่ละ commit โดยใช้ SHA-1 hash เป็น ID เฉพาะ

🔒
SHA-1 Hash คืออะไร? ตัวเลขฐาน 16 ยาว 40 ตัวอักษร เช่น a1b2c3d4e5f6...
ถ้า content ของไฟล์เปลี่ยนแม้แต่ 1 ตัวอักษร — hash จะเปลี่ยนทั้งหมด ทำให้ตรวจสอบ integrity ได้

Object Types ใน Git:

Objectคืออะไรตัวอย่าง
blobเนื้อหาของไฟล์หนึ่งไฟล์เนื้อหาของ index.js
treeโครงสร้าง directory (ชี้ไป blobs และ trees อื่น)รายการไฟล์ใน /src
commitชี้ไป tree + parent commit + metadataSnapshot ณ เวลา commit
tagชี้ไป commit พร้อมชื่อและ messagev1.0.0
🌿4. Branching — Branch คือแค่ Pointer

Branch ใน Git ไม่ใช่การ copy ไฟล์ — มันคือ pointer (ตัวชี้) ที่ชี้ไปยัง commit ล่าสุด เมื่อเราสร้าง commit ใหม่ pointer จะขยับตามโดยอัตโนมัติ

C1 ──── C2 ──── C3 ◄── main │ └── C4 ──── C5 ◄── feature/login ▲ HEAD (อยู่ที่นี่ตอนนี้)

HEAD คือ pointer ที่บอกว่าเราอยู่ที่ branch/commit ไหนตอนนี้ ปกติ HEAD ชี้ไปที่ branch ล่าสุด

💡
ทำไม Git branch ถึงเร็วมาก? เพราะการสร้าง branch แค่สร้าง file เล็กๆ ที่เก็บ SHA-1 ของ commit ล่าสุด — ไม่ต้อง copy ไฟล์ใดเลย จึงใช้เวลาเพียงไม่กี่ milliseconds
🔀5. Merge vs Rebase — ความแตกต่าง
git merge (Non-destructive)
รวม commits โดยสร้าง "Merge Commit" ใหม่ — ประวัติที่แท้จริงจะยังอยู่ครบ เห็นว่า branches รวมกันตรงไหน เหมาะกับ team workflow
git rebase (Clean History)
ย้าย commits ไปเริ่มต้นใหม่จาก tip ของอีก branch — ประวัติดูตรงเป็นเส้นตรง อ่านง่ายกว่า แต่ ห้ามใช้กับ commits ที่ push ไปแล้ว
⚠️
กฎทอง: ห้าม rebase commit ที่ push ขึ้น remote แล้ว เพราะมันเขียน history ใหม่ — คนอื่นที่ดึงไปแล้วจะเกิด conflict ที่แก้ยาก ใช้ rebase ได้แค่กับ local commits ที่ยังไม่ push
⚙️6. Basic Git Workflow ทีละขั้นตอน
1
Clone หรือ Init
git clone URL — ดึง repo มาจาก remote หรือ git init — สร้าง repo ใหม่บนเครื่อง
2
แก้ไขไฟล์
แก้ไขไฟล์ด้วย VS Code ตามปกติ — ไฟล์จะอยู่ใน "Working Tree" และมีสถานะ Modified
3
Stage การเปลี่ยนแปลง
git add filename หรือ git add . (ทั้งหมด) — ย้ายไปยัง Staging Area เพื่อเตรียม commit
4
Commit
git commit -m "message" — บันทึก snapshot ลง Local Repository พร้อม message อธิบาย
5
Push
git push origin main — ส่ง commits จาก Local ขึ้น Remote Repository (GitHub)
6
Pull
git pull origin main — ดึง commits ล่าสุดจาก Remote มาที่ Local (= fetch + merge)
⌨️7. คำสั่ง Git ที่ต้องรู้
git status
ดูสถานะไฟล์ทั้งหมด
git log --oneline
ดู commit history แบบย่อ
git diff
ดูการเปลี่ยนแปลงที่ยังไม่ stage
git checkout -b name
สร้าง branch ใหม่และสลับไป
git stash
เก็บงานค้างไว้ชั่วคราว
git stash pop
นำงานที่เก็บไว้กลับมา
git reset HEAD~1
ยกเลิก commit ล่าสุด (ไฟล์ยังอยู่)
git blame filename
ดูว่าใคร commit บรรทัดไหน
📖8. คำศัพท์ที่ควรรู้
repository
ที่เก็บ code พร้อมประวัติ commits ทั้งหมด — มีทั้ง local และ remote
commit
จุดบันทึก snapshot ของ code ณ เวลาหนึ่ง มี ID เป็น SHA-1 hash
branch
pointer ที่ชี้ไป commit ล่าสุดของสาย development นั้น
HEAD
pointer พิเศษที่บอกว่า "คุณอยู่ที่ branch/commit ไหนตอนนี้"
origin
ชื่อ default ของ remote repository — ตั้งชื่ออื่นได้แต่ convention คือ origin
upstream
remote branch ที่ local branch ติดตามอยู่ ใช้ใน git push -u origin main
merge conflict
เกิดเมื่อ 2 branches แก้ไฟล์บรรทัดเดิม — ต้องแก้ด้วยตนเองก่อน commit
.gitignore
ไฟล์กำหนดว่าไฟล์/folder ไหนที่ Git จะไม่ track เช่น node_modules, .env
