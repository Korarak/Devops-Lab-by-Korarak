Database Schema + Seed Data
Database

ตาราง tasks ออกแบบมาให้เก็บงานได้ครบทุก attribute — สร้างอัตโนมัติตอน backend start ครั้งแรก

SQL Schema: backend/index.js → initDb()
CREATE TABLE tasks
CREATE TABLE IF NOT EXISTS tasks (
  id          SERIAL PRIMARY KEY,
  title       VARCHAR(200) NOT NULL,
  status      VARCHAR(20)  NOT NULL DEFAULT 'todo',
  priority    VARCHAR(10)  NOT NULL DEFAULT 'medium',
  assignee    VARCHAR(100) DEFAULT '',
  due_date    DATE,
  description TEXT         DEFAULT '',
  created_at  TIMESTAMPTZ  DEFAULT NOW(),
  updated_at  TIMESTAMPTZ  DEFAULT NOW()
)

อธิบายแต่ละ column:

id SERIAL PRIMARY KEYauto-increment integer — ทำหน้าที่เป็น unique identifier ของแต่ละงาน
title VARCHAR(200) NOT NULLชื่องาน — บังคับต้องมี, ยาวสูงสุด 200 ตัวอักษร
status VARCHAR(20) DEFAULT 'todo'สถานะงาน — ค่าที่ถูกต้อง: todo / in-progress / done (validate ใน API)
priority VARCHAR(10) DEFAULT 'medium'ระดับความสำคัญ — ค่าที่ถูกต้อง: low / medium / high (validate ใน API)
assignee VARCHAR(100)ชื่อผู้รับผิดชอบ — optional, default ว่าง
due_date DATEวันกำหนดส่ง — NULL ได้ถ้าไม่ระบุ, ใช้เปรียบเทียบใน isOverdue
description TEXTรายละเอียดงาน — TEXT รองรับข้อความยาวได้ไม่จำกัด
created_at / updated_at TIMESTAMPTZtimestamp พร้อม timezone — DEFAULT NOW() กำหนดอัตโนมัติ

GET /api/stats — FILTER aggregate query:

index.js — Stats endpoint (single query)
app.get('/api/stats', async (req, res) => {
  try {
    const { rows } = await pool.query(`
      SELECT
        COUNT(*)                                                       AS total,
        COUNT(*) FILTER (WHERE status = 'todo')                        AS todo,
        COUNT(*) FILTER (WHERE status = 'in-progress')                 AS in_progress,
        COUNT(*) FILTER (WHERE status = 'done')                        AS done,
        COUNT(*) FILTER (WHERE due_date < NOW() AND status != 'done')  AS overdue,
        COUNT(*) FILTER (WHERE priority = 'high' AND status != 'done') AS high_priority
      FROM tasks
    `);
    res.json(rows[0]);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});
FILTER (WHERE ...)PostgreSQL aggregate filter — นับเฉพาะ rows ที่ match condition โดยไม่ต้องทำหลาย query
single queryได้ stats ทั้งหมด 6 ค่าใน 1 round trip แทนที่จะ query 6 ครั้ง — ประสิทธิภาพดีกว่ามาก
due_date < NOW() AND status != 'done'งาน overdue = ผ่านวันกำหนดส่งแล้ว แต่ยังไม่ done

GET /api/tasks — Dynamic WHERE builder:

index.js — Tasks list (3 filters)
app.get('/api/tasks', async (req, res) => {
  try {
    const { search = '', status = '', priority = '' } = req.query;
    const params = [];
    let where = 'WHERE 1=1';

    if (search) {
      params.push(`%${search}%`);
      where += ` AND (title ILIKE $${params.length} OR description ILIKE $${params.length})`;
    }
    if (status) {
      params.push(status);
      where += ` AND status = $${params.length}`;
    }
    if (priority) {
      params.push(priority);
      where += ` AND priority = $${params.length}`;
    }

    const { rows } = await pool.query(
      `SELECT * FROM tasks ${where} ORDER BY created_at DESC`,
      params
    );
    res.json(rows);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});
WHERE 1=1เริ่มต้น condition ที่เป็น true เสมอ — ทำให้ append AND ต่อท้ายได้ทุก filter
ILIKE $1case-insensitive LIKE ค้นหาใน title หรือ description — $1 คือ parameterized query ป้องกัน SQL Injection
3 filterssearch + status + priority — ซับซ้อนกว่า StockPro ที่มีแค่ search + category

POST /api/tasks — Validation Logic (สำคัญสำหรับ Jest test):

index.js — POST validation
app.post('/api/tasks', async (req, res) => {
  try {
    const { title, status = 'todo', priority = 'medium',
            assignee = '', due_date = null, description = '' } = req.body;

    // Validation 1: title ต้องมีค่า
    if (!title || !title.toString().trim()) {
      return res.status(400).json({ error: 'กรุณาระบุ title' });
    }

    const VALID_STATUS   = ['todo', 'in-progress', 'done'];
    const VALID_PRIORITY = ['low', 'medium', 'high'];

    // Validation 2: status ต้องเป็นค่าที่กำหนด (ถ้าส่งมา)
    if (req.body.status !== undefined && !VALID_STATUS.includes(status)) {
      return res.status(400).json({ error: `status ต้องเป็น ${VALID_STATUS.join(', ')}` });
    }

    // Validation 3: priority ต้องเป็นค่าที่กำหนด (ถ้าส่งมา)
    if (req.body.priority !== undefined && !VALID_PRIORITY.includes(priority)) {
      return res.status(400).json({ error: `priority ต้องเป็น ${VALID_PRIORITY.join(', ')}` });
    }

    const { rows } = await pool.query(
      `INSERT INTO tasks (title, status, priority, assignee, due_date, description)
       VALUES ($1, $2, $3, $4, $5, $6) RETURNING *`,
      [title.toString().trim(), status, priority, assignee, due_date || null, description]
    );
    res.status(201).json(rows[0]);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});
!title || !title.toString().trim()ตรวจทั้ง null/undefined และ empty string หลัง trim — ป้องกัน title ที่เป็น spaces
req.body.status !== undefinedตรวจว่า client ส่ง status มาจริงๆ — ถ้าไม่ส่ง ใช้ default 'todo' โดยไม่ validate
VALID_STATUS.includes(status)whitelist validation — ค่าที่ไม่อยู่ใน array จะ return 400 พร้อม error message ภาษาไทย
res.status(400)HTTP 400 Bad Request — validation.test.js ใน Jest จะ assert ว่าได้ 400 กลับมา
🧪
ทำไม Validation ถึงสำคัญสำหรับ Testing? Jest ใน validation.test.js จะส่ง request ที่ผิดแล้วตรวจว่าได้ response 400 กลับมา
ถ้า validation logic ชัดเจน → เขียน test ได้ง่าย → coverage ผ่าน threshold 60%
