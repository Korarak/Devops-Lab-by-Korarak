เตรียม Tools ให้พร้อม
ตรวจสอบก่อนเริ่ม
⏱ 5 นาที
📚 อ้างอิง Week 1 — Git Fundamentals

ตรวจสอบว่าติดตั้ง tools ครบก่อนเริ่ม เปิด Terminal/PowerShell แล้วพิมพ์ทีละคำสั่ง:

Terminal
git --version       # ต้องได้ git version 2.x.x
node --version      # ต้องได้ v18.x.x หรือสูงกว่า
npm --version       # ต้องได้ 9.x.x หรือสูงกว่า
docker --version    # ต้องได้ Docker version 24.x.x หรือสูงกว่า
docker compose version  # ต้องได้ v2.x.x
💡
Docker Desktop ต้องเปิดอยู่ ตรวจสอบ icon Docker ที่ Taskbar ถ้าไม่มี ให้เปิดโปรแกรม Docker Desktop ก่อน แล้วรอจน icon เปลี่ยนเป็น Engine running
⚠️
ต้องมี Docker Hub Account ถ้ายังไม่มี ให้สมัครที่ hub.docker.com ก่อน จด username ไว้ จะใช้ใน Part 3–4

3
Clone และสร้าง Folder Structure
clone repo + สร้างโฟลเดอร์โปรเจค
⏱ 5 นาที
📚 อ้างอิง Week 1 — git clone

เปิด Terminal แล้วเลือกที่เก็บโปรเจค (เช่น D:\projects) แล้วรัน:

Terminal
git clone https://github.com/<your-username>/taskboard.git
cd taskboard

สร้างโครงสร้างโฟลเดอร์ด้วยคำสั่งเดียว:

Terminal (Windows PowerShell)
# สร้างโฟลเดอร์ทั้งหมดพร้อมกัน
mkdir backend\src\controllers, backend\src\routes
mkdir frontend\src\components
mkdir .github\workflows
Terminal (macOS/Linux)
mkdir -p backend/src/{controllers,routes}
mkdir -p frontend/src/components
mkdir -p .github/workflows
💜
แนะนำ: เปิดด้วย VS Code พิมพ์ code . ใน terminal เพื่อเปิดโฟลเดอร์ทั้งหมดใน VS Code ได้เลย
4
สร้าง .gitignore
ป้องกันไม่ให้ push ไฟล์ที่ไม่ควร
⏱ 5 นาที
📚 อ้างอิง Week 1 — .gitignore

แก้ไขไฟล์ .gitignore ที่ root ของโปรเจค (GitHub สร้างให้แล้ว) ให้ครอบคลุมทั้ง backend และ frontend:

.gitignore
# Dependencies
node_modules/

# Environment variables — ห้าม commit เด็ดขาด!
.env
.env.local

# Build output
dist/
build/

# Docker
*.log

# OS files
.DS_Store
Thumbs.db
⚠️
สำคัญมาก: ห้าม commit ไฟล์ .env ไฟล์ .env เก็บ password ฐานข้อมูล ถ้า push ขึ้น GitHub สาธารณะ จะถูก bot ขโมยข้อมูลภายใน 30 นาที!
5
สร้าง README.md
เอกสารโปรเจค
⏱ 5 นาที
📚 อ้างอิง Week 1 — GitHub Profile & README

แก้ไขไฟล์ README.md ให้มีเนื้อหาพื้นฐาน (CI badge จะเพิ่มใน Part 4):

README.md
# TaskBoard — ระบบจัดการงาน

![CI](https://github.com/<username>/taskboard/actions/workflows/ci.yml/badge.svg)

ระบบ CRUD สำหรับจัดการงาน สร้างด้วย Express.js + PostgreSQL + Vue 3

## Tech Stack
- **Backend:** Node.js + Express.js + PostgreSQL
- **Frontend:** Vue 3 + Vite
- **Container:** Docker + Docker Compose
- **CI/CD:** GitHub Actions

## วิธีรัน (Local)
```bash
# Clone
git clone https://github.com/<username>/taskboard.git
cd taskboard

# รัน ด้วย Docker Compose
docker compose up --build
```
เข้า http://localhost เพื่อดู Frontend

## API Endpoints
| Method | Path | คำอธิบาย |
|--------|------|----------|
| GET | /health | Health check |
| GET | /api/tasks | ดึงงานทั้งหมด |
| POST | /api/tasks | สร้างงานใหม่ |
| PUT | /api/tasks/:id | แก้ไขงาน |
| DELETE | /api/tasks/:id | ลบงาน |
6
Initial Commit & Push
บันทึกงานขึ้น GitHub ครั้งแรก
⏱ 5 นาที
📚 อ้างอิง Week 1 — git add / commit / push

บันทึกการเปลี่ยนแปลงและ push ขึ้น GitHub:

Terminal
git add .gitignore README.md
git commit -m "feat: initial project structure"
git push origin main

ตรวจสอบผลบน GitHub โดยเปิด https://github.com/<username>/taskboard

💡
Commit Message Convention ใช้รูปแบบ feat:, fix:, docs:, chore: นำหน้า เพื่อให้อ่านง่าย — เป็น Conventional Commits ที่ใช้ในอุตสาหกรรม
7
สร้าง Branch develop และ feature
จัดการ branch ตาม GitFlow
⏱ 5 นาที
📚 อ้างอิง Week 1 — Branching Strategy

โปรเจคนี้ใช้ GitFlow แบบย่อ: main → develop → feature/*

main ──●────────────────────────●── (stable)
develop ╲──────────────────●╱ (integration)
feature/backend ╲──────●╱ (your work)
Terminal
# สร้าง develop branch แล้ว push
git checkout -b develop
git push -u origin develop

# สร้าง feature branch สำหรับ Part 2
git checkout -b feature/backend-setup
# (ตอนนี้อยู่บน feature/backend-setup แล้ว)
✅
ตรวจเช็ค branch บน GitHub ไปที่ GitHub repo → ดูที่ dropdown "main ▼" ควรเห็น: main, develop, feature/backend-setup
