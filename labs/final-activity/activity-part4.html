📚 อ้างอิง Week 4 — GitHub Secrets

Secrets คือ environment variables ที่เก็บใน GitHub อย่างปลอดภัย — ไม่แสดงใน log ไม่ต้อง hardcode ใน code

ขั้นตอน 1: สร้าง Docker Hub Access Token

  1. เข้า hub.docker.com → Account Settings
  2. เลือก Security → New Access Token
  3. ตั้งชื่อ: github-actions-taskboard
  4. Permissions: Read, Write, Delete
  5. คลิก Generate แล้วคัดลอก token ไว้ (แสดงครั้งเดียว!)

ขั้นตอน 2: เพิ่ม Secrets ใน GitHub Repository

  1. ไปที่ GitHub repo taskboard
  2. คลิก Settings → Secrets and variables → Actions
  3. คลิก New repository secret แล้วเพิ่มครบทั้งสอง:
Secret NameValueใช้ทำอะไร
DOCKERHUB_USERNAME Docker Hub username ของคุณ ชื่อ user สำหรับ login
DOCKERHUB_TOKEN Access Token ที่สร้างในขั้นตอนก่อน รหัสผ่าน (ปลอดภัยกว่า password)
⚠️
ใช้ Access Token ไม่ใช่ Password! Docker Hub แนะนำ Access Token เพราะสามารถ revoke ได้ทีละ token และกำหนด scope ได้ ถ้า token รั่วก็ลบแค่ token นั้น ไม่ต้องเปลี่ยน password ทั้งหมด

สร้าง .github/workflows/ci.yml
CI: Lint + Build + Docker Build Test
⏱ 15 นาที
📚 อ้างอิง Week 4 — GitHub Actions Workflow YAML

สร้างไฟล์ .github/workflows/ci.yml:

.github/workflows/ci.yml
name: CI — TaskBoard

# ────────────────────────────────────────────────
# Trigger: push ไปยัง feature/* หรือ develop
#          หรือ PR เข้า develop/main
# ────────────────────────────────────────────────
on:
  push:
    branches:
      - develop
      - 'feature/**'
  pull_request:
    branches:
      - develop
      - main

jobs:
  lint-and-build:
    name: Lint & Build
    runs-on: ubuntu-latest   # ใช้ GitHub-hosted runner (Ubuntu)

    steps:
      # Step 1: ดึง source code จาก GitHub มาที่ Runner
      - name: Checkout code
        uses: actions/checkout@v4

      # Step 2: ติดตั้ง Node.js เวอร์ชัน 20
      - name: Setup Node.js 20
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'             # cache node_modules ให้ build เร็วขึ้น
          cache-dependency-path: backend/package-lock.json

      # ── Backend ──────────────────────────────
      - name: Install backend dependencies
        working-directory: backend   # เปลี่ยน directory ก่อนรัน
        run: npm ci                  # ci = clean install (เร็วกว่า npm install)

      - name: Lint backend code
        working-directory: backend
        run: npm run lint            # ถ้า ESLint error → job fail → CI ไม่ผ่าน

      # ── Frontend ──────────────────────────────
      - name: Install frontend dependencies
        working-directory: frontend
        run: npm ci

      - name: Build frontend (production)
        working-directory: frontend
        run: npm run build           # สร้าง dist/ — ถ้า build fail → CI ไม่ผ่าน

      # ── Docker Smoke Test ─────────────────────
      - name: Docker build test (backend)
        run: docker build -t taskboard-backend:ci ./backend

      - name: Docker build test (frontend)
        run: docker build -t taskboard-frontend:ci ./frontend
💜
อธิบายแต่ละ keyword ใน YAML: name: ชื่อที่แสดงใน GitHub Actions UI
on: เงื่อนไข trigger
jobs: กลุ่มงาน (parallel ได้)
steps: ลำดับขั้นตอนใน job
uses: ใช้ Action สำเร็จรูปจาก GitHub Marketplace
run: รัน shell command

สร้าง .github/workflows/cd.yml
CD: Build & Push to Docker Hub
⏱ 10 นาที
📚 อ้างอิง Week 4 — CD Pipeline + Docker Hub Push

สร้างไฟล์ .github/workflows/cd.yml:

.github/workflows/cd.yml
name: CD — Deploy TaskBoard

# ────────────────────────────────────────────────
# Trigger: push ไปยัง main เท่านั้น (หลัง PR merge)
# ────────────────────────────────────────────────
on:
  push:
    branches:
      - main

jobs:
  build-and-push:
    name: Build & Push to Docker Hub
    runs-on: ubuntu-latest

    steps:
      # Step 1: ดึง source code
      - name: Checkout code
        uses: actions/checkout@v4

      # Step 2: Login เข้า Docker Hub ด้วย Secret
      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}   # ดึงจาก GitHub Secrets
          password: ${{ secrets.DOCKERHUB_TOKEN }}      # ไม่มีใครเห็นค่านี้ใน log

      # Step 3: Build และ Push Backend Image
      - name: Build & push backend image
        uses: docker/build-push-action@v5
        with:
          context: ./backend          # ใช้ backend/Dockerfile
          push: true                  # push ขึ้น registry จริง
          tags: |
            ${{ secrets.DOCKERHUB_USERNAME }}/taskboard-backend:latest

      # Step 4: Build และ Push Frontend Image
      - name: Build & push frontend image
        uses: docker/build-push-action@v5
        with:
          context: ./frontend
          push: true
          tags: |
            ${{ secrets.DOCKERHUB_USERNAME }}/taskboard-frontend:latest
💡
docker/build-push-action@v5 ดีกว่า docker build + docker push อย่างไร? Action นี้รองรับ BuildKit (build เร็วกว่า), multi-platform build, และ cache layer โดยอัตโนมัติ — ประหยัดเวลาใน CI

Commit workflow ใหม่:

Terminal
git add .github/workflows/cd.yml
git commit -m "feat: add CD workflow (build & push to Docker Hub)"
git push origin feature/cicd

📚 อ้างอิง Week 4 — GitFlow: feature → develop → main

ทำตามขั้นตอน GitFlow:

  1. เปิด PR: feature/cicd → develop บน GitHub แล้ว merge
  2. เปิด PR: develop → main แล้ว merge
  3. ดู tab Actions — จะเห็น workflow "CD — Deploy TaskBoard" เริ่ม run
  4. รอจน CD ผ่าน (ประมาณ 2–5 นาที)
✅
ยืนยัน CD ทำงาน: เข้า Docker Hub → Repositories → taskboard-backend และ taskboard-frontend → ดู "Last pushed" ต้องเป็นเวลาล่าสุด
