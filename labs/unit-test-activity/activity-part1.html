Unit Testing คืออะไร? และทำไมต้องใช้
⏱ 8 นาที

Unit Test คือการทดสอบ "หน่วยย่อยที่เล็กที่สุด" ของโค้ด — โดยทั่วไปคือ function เดียว หรือ method เดียว โดยแยกออกจากส่วนอื่นๆ ของโปรแกรม

❌ ไม่มี Unit Test

แก้โค้ด → รันโปรแกรมทั้งหมด → ทดสอบด้วยมือ → หวังว่าจะไม่พัง → พบ bug ที่ production

✅ มี Unit Test

แก้โค้ด → รัน Test อัตโนมัติ → รู้ทันทีว่า function ไหนพัง → แก้ก่อน merge

ทำไมต้องเขียน Unit Test?

  • ป้องกัน bug: ตรวจพบปัญหาก่อนที่จะถึง production
  • Refactor ได้อย่างมั่นใจ: เปลี่ยนโค้ดแล้วรัน test → รู้ทันทีว่าอะไรพัง
  • เป็น Documentation ที่รันได้: Test บอกว่า function นี้ "ควรทำอะไร" อย่างชัดเจน
  • Debug ง่ายขึ้น: เมื่อ test fail รู้ทันทีว่า function ไหนมีปัญหา
  • ความมั่นใจในการ Deploy: ทีมงานมั่นใจว่าโค้ดทำงานถูกต้องก่อน deploy

เปรียบเทียบให้เข้าใจง่าย

นึกภาพการผลิตรถยนต์ — วิศวกรจะ ทดสอบอะไหล่แต่ละชิ้น ก่อนประกอบ:

  • ทดสอบเบรก → ต้องหยุดได้ในระยะ X เมตร
  • ทดสอบเครื่องยนต์ → ต้องได้แรงม้า X แรงม้า
  • ทดสอบไฟ → ต้องสว่างได้ X ลูเมน

Unit Test ก็คือการทดสอบ "อะไหล่" แต่ละชิ้นของโค้ดเช่นกัน

💡
หลักการสำคัญUnit Test ที่ดีต้องทดสอบ function เดียว, ทำงาน เร็วมาก (ไม่กี่มิลลิวินาที), และ ไม่ขึ้นกับสิ่งภายนอก เช่น database, network หรือ file system

Testing Pyramid — Unit Test อยู่ตรงไหน?
⏱ 7 นาที

Testing Pyramid อธิบายความสัมพันธ์ระหว่างการทดสอบ 3 ประเภท โดยจัดเรียงตาม ความเร็ว, ต้นทุน และ จำนวน:

E2E Tests น้อย · ช้า · แพง
Integration Tests ปานกลาง
Unit Tests ← เราอยู่นี่ เยอะสุด · เร็วสุด · ถูกสุด

ฐานกว้างสุด = Unit Tests มากที่สุด

Unit Tests (ฐาน)

  • ทดสอบ function เดียว
  • รันใน < 1 วินาที
  • ไม่ต้องการ DB หรือ Network
  • เขียนเยอะที่สุด (70-80%)

Integration Tests (กลาง)

  • ทดสอบหลาย component ร่วมกัน
  • อาจใช้ DB หรือ API จริง
  • ช้ากว่า unit test
  • เขียนปานกลาง (15-20%)

E2E Tests (ยอด)

  • จำลองการใช้งานจริงของ user
  • ช้ามาก อาจใช้ browser จริง
  • เปราะบาง (flaky) ง่าย
  • เขียนน้อยที่สุด (5-10%)
🎯
เป้าหมายกิจกรรมนี้เรียนรู้การเขียน Unit Tests และ Integration Tests ระดับเบื้องต้น (การทดสอบ API endpoint ด้วย Supertest)

ตั้งค่าโปรเจกต์ ScoreCalc + ติดตั้ง Jest
⏱ 10 นาที

เราจะสร้างโปรเจกต์ ScoreCalc ซึ่งเป็น API คำนวณคะแนนและเกรดนักเรียน — เหมาะสำหรับเริ่มเรียนรู้การเขียน Test เพราะ business logic ชัดเจนและเข้าใจง่าย

1. สร้างโปรเจกต์

Terminal
mkdir scorecalc
cd scorecalc
npm init -y
npm install express
npm install --save-dev jest supertest

2. โครงสร้างโฟลเดอร์ที่ควรได้

scorecalc/ ├── src/ │ ├── utils/ │ │ └── calculator.js ← ฟังก์ชันคำนวณ (จะสร้างใน Step 4) │ └── app.js ← Express app (จะสร้างใน Part 3) ├── tests/ │ └── calculator.test.js ← Test file (จะสร้างใน Step 4) ├── node_modules/ └── package.json
Terminal — สร้างโฟลเดอร์
mkdir -p src/utils tests

3. ตั้งค่า package.json

เปิดไฟล์ package.json แล้วเพิ่ม/แก้ไขส่วน scripts และเพิ่ม jest config:

📌
อธิบาย scripts
  • npm test → รัน Jest ครั้งเดียว
  • npm run test:watch → รัน Jest แบบ watch mode (รันใหม่ทุกครั้งที่บันทึกไฟล์)
  • npm run test:coverage → รันพร้อมสร้าง coverage report
✅
ตรวจสอบ
  • โฟลเดอร์ src/utils/ และ tests/ ถูกสร้างแล้ว
  • package.json มี "test": "jest" ใน scripts
  • node_modules/jest มีอยู่ (หลัง npm install)
4
เขียน Test แรก — ฟังก์ชัน Calculator
⏱ 12 นาที

เราจะสร้าง 2 ไฟล์: (1) ไฟล์โค้ดจริง และ (2) ไฟล์ test — นี่คือ pattern พื้นฐานของ Unit Testing

1. สร้างไฟล์โค้ดจริง: src/utils/calculator.js

src/utils/calculator.js
// ฟังก์ชันคำนวณพื้นฐาน
function add(a, b) {
  return a + b;
}

function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

function divide(a, b) {
  return a / b;
}

// ฟังก์ชันคำนวณคะแนนเฉลี่ย
function average(scores) {
  if (scores.length === 0) return 0;
  const sum = scores.reduce((acc, val) => acc + val, 0);
  return sum / scores.length;
}

// ฟังก์ชันแปลงคะแนนเป็นเกรด
function getGrade(score) {
  if (score >= 80) return 'A';
  if (score >= 70) return 'B';
  if (score >= 60) return 'C';
  if (score >= 50) return 'D';
  return 'F';
}

module.exports = { add, subtract, multiply, divide, average, getGrade };

2. สร้างไฟล์ Test: tests/calculator.test.js

ชื่อไฟล์ test ต้องลงท้ายด้วย .test.js หรือ .spec.js — Jest จะหาไฟล์เหล่านี้อัตโนมัติ

tests/calculator.test.js
// import ฟังก์ชันที่ต้องการทดสอบ
const { add, subtract, multiply, divide, average, getGrade } = require('../src/utils/calculator');

// test() รับ 2 argument:
//   1. ชื่อ Test (อธิบายว่าทดสอบอะไร)
//   2. ฟังก์ชัน callback ที่มีโค้ด test

test('add: 2 + 3 ต้องได้ 5', () => {
  expect(add(2, 3)).toBe(5);
});

test('add: 0 + 0 ต้องได้ 0', () => {
  expect(add(0, 0)).toBe(0);
});

test('subtract: 10 - 4 ต้องได้ 6', () => {
  expect(subtract(10, 4)).toBe(6);
});

test('multiply: 3 × 4 ต้องได้ 12', () => {
  expect(multiply(3, 4)).toBe(12);
});

test('divide: 15 ÷ 3 ต้องได้ 5', () => {
  expect(divide(15, 3)).toBe(5);
});

test('average: คะแนน [80, 90, 70] ต้องได้เฉลี่ย 80', () => {
  expect(average([80, 90, 70])).toBe(80);
});

test('average: array ว่างต้องได้ 0', () => {
  expect(average([])).toBe(0);
});

test('getGrade: คะแนน 85 ต้องได้เกรด A', () => {
  expect(getGrade(85)).toBe('A');
});

test('getGrade: คะแนน 72 ต้องได้เกรด B', () => {
  expect(getGrade(72)).toBe('B');
});

test('getGrade: คะแนน 45 ต้องได้เกรด F', () => {
  expect(getGrade(45)).toBe('F');
});
🧠
โครงสร้างพื้นฐานของ Test
  • test('ชื่อ', () => { ... }) — กำหนด test case
  • expect(ผลลัพธ์) — บอกว่าต้องการตรวจสอบอะไร
  • .toBe(ค่าที่คาดหวัง) — บอกว่าคาดว่าจะได้อะไร

// แก้ให้ผิดชั่วคราว
function add(a, b) {
  return a * b;  // Bug: คูณแทนที่จะบวก!
