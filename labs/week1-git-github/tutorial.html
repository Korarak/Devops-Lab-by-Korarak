1
ติดตั้ง Git for Windows
git-scm.com ▼
  1. เปิด Browser → ไปที่ https://git-scm.com/download/win
  2. คลิก "64-bit Git for Windows Setup" → รอดาวน์โหลด → รัน installer
  3. ตั้งค่าระหว่าง install เลือกค่าเหล่านี้ (ค่าอื่นปล่อย default):
    • Default editor → เลือก "Use Visual Studio Code"
    • Adjusting PATH → เลือก "Git from the command line and also 3rd-party software"
    • Line ending → เลือก "Checkout Windows-style, commit Unix-style"
  4. เมื่อ install เสร็จ → เปิด PowerShell ใหม่ → พิมพ์คำสั่งด้านล่าง
PowerShell
git --version
✅ ผลที่ควรเห็น
  • git version 2.4x.x (ตัวเลขอาจต่างกันเล็กน้อย)
2
ติดตั้ง Node.js (LTS)
nodejs.org ▼
  1. ไปที่ https://nodejs.org → คลิกปุ่ม "LTS" สีเขียว → ดาวน์โหลดไฟล์ .msi
  2. รัน installer → คลิก Next ทุกขั้น (ไม่ต้องเปลี่ยนอะไร) → Finish
  3. เปิด PowerShell ใหม่ → ทดสอบ:
PowerShell
node --version
npm --version
✅ ผลที่ควรเห็น
  • Node.js: v20.x.x หรือสูงกว่า
  • npm: 10.x.x
3
ติดตั้ง Visual Studio Code
code.visualstudio.com ▼
  1. ไปที่ https://code.visualstudio.com → คลิก "Download for Windows"
  2. รัน installer → ติ๊ก "Add to PATH" และ "Open with Code" ทั้งสองช่อง → Install
  3. เปิด VS Code → กด Ctrl+` (backtick) เพื่อเปิด Terminal ด้านล่าง
💡
Extension แนะนำ: กด Ctrl+Shift+X แล้วค้นหาและติดตั้ง GitLens เพื่อดู git history ได้สะดวกขึ้น
✅ ผลที่ควรเห็น
  • VS Code เปิดได้และมี Terminal ด้านล่าง

ติดตั้ง Docker Desktop
docker.com ▼
  1. ไปที่ https://www.docker.com/products/docker-desktop → คลิก "Download for Windows"
  2. รัน installer → เมื่อถามเรื่อง WSL 2 ให้ติ๊ก "Use WSL 2 instead of Hyper-V" → OK
⚠️
ระวัง: ถ้าเครื่องขอ Restart → บันทึกงานทั้งหมดก่อน แล้วค่อย Restart
  1. หลัง Restart → เปิด Docker Desktop → รอจนไอคอน Docker ล่างขวาเป็น สีเขียว (Running)
  2. ทดสอบใน PowerShell:
PowerShell
docker --version
docker run hello-world
✅ ผลที่ควรเห็น
  • ข้อความ "Hello from Docker!" ในผลลัพธ์

github.com ▼

ตั้งค่า Git บนเครื่อง (Global Config)
▼

เปิด PowerShell → พิมพ์คำสั่งด้านล่าง แทนที่ข้อมูลในเครื่องหมาย " " ด้วยชื่อและ email จริงของคุณ

PowerShell
git config --global user.name "Your Name in English"
git config --global user.email "your-email@github.com"
git config --global core.autocrlf true

# ตรวจสอบว่าบันทึกแล้ว
git config --global --list
✅ ผลที่ควรเห็น
  • เห็น user.name=Your Name ในผลลัพธ์
  • เห็น user.email=your-email@github.com
7
สร้าง SSH Key และผูกกับ GitHub
▼

ขั้นตอน 1 — สร้าง SSH Key

เปิด PowerShell → พิมพ์คำสั่งด้านล่าง กด Enter ทุกครั้งที่ถาม (ไม่ต้องใส่ passphrase)

PowerShell — แทนที่ email ด้วย email ที่ใช้สมัคร GitHub
ssh-keygen -t ed25519 -C "your-email@github.com"

ขั้นตอน 2 — คัดลอก Public Key

PowerShell
cat $HOME\.ssh\id_ed25519.pub
ℹ️
ผลที่ได้จะเป็นข้อความยาวขึ้นต้นด้วย ssh-ed25519 ... — เลือกและคัดลอกทั้งหมด

ขั้นตอน 3 — เพิ่ม Key บน GitHub

  1. ไปที่ GitHub → คลิกรูปโปรไฟล์มุมบนขวา → Settings
  2. เลือก SSH and GPG keys → คลิก New SSH key
  3. ตั้งชื่อ: "School PC" → วางข้อความที่คัดลอกมา → คลิก "Add SSH key"

ขั้นตอน 4 — ทดสอบการเชื่อมต่อ

PowerShell
ssh -T git@github.com
⚠️
ถ้าเห็นข้อความ "Are you sure you want to continue connecting?" → พิมพ์ yes แล้วกด Enter
✅ ผลที่ควรเห็น
  • Hi username! You've successfully authenticated, but GitHub does not provide shell access.

สร้าง Local Repository แรก
▼

สร้างโฟลเดอร์ทดลอง → เข้า VS Code Terminal แล้วพิมพ์:

VS Code Terminal / PowerShell
# สร้างโฟลเดอร์
mkdir C:\devops-labs\git-practice
cd C:\devops-labs\git-practice

# เริ่มต้น Git repository
git init

# ดูสถานะ (ยังว่างอยู่)
git status
✅ ผลที่ควรเห็น
  • Initialized empty Git repository in C:/devops-labs/git-practice/.git/
  • เห็น On branch master หรือ main
9
สร้างไฟล์และทำ Commit แรก
▼
VS Code Terminal
# สร้างไฟล์
echo "# Git Practice" > README.md

# ดู status — ไฟล์ยัง untracked (สีแดง)
git status

# เพิ่มไฟล์เข้า staging area
git add README.md

# ดู status อีกครั้ง — ไฟล์เป็นสีเขียว (staged)
git status

# บันทึก commit แรก
git commit -m "docs: add README"

# ดูประวัติ
git log --oneline
💡
รูปแบบ Commit Message ที่ดี (Conventional Commits):
feat: เพิ่ม feature ใหม่  |  fix: แก้บัก  |  docs: แก้เอกสาร  |  chore: งานทั่วไป
ตัวอย่าง: feat: add login page, fix: correct SQL query
✅ ผลที่ควรเห็น
  • เห็น commit แรกใน git log --oneline
  • message เป็น docs: add README
10
ฝึก Branch และ Merge
▼
VS Code Terminal
# ดู branch ปัจจุบัน
git branch

# สร้าง branch ใหม่และเปลี่ยนไปใช้
git checkout -b feature/add-description

# แก้ไข README.md ใน VS Code — เพิ่มบรรทัดนี้ต่อท้าย:
# "นักเรียน: ชื่อของคุณ"

# บันทึก
git add README.md
git commit -m "docs: add student name to README"

# กลับไป main (หรือ master)
git checkout main

# รวม feature branch เข้า main
git merge feature/add-description

# ดูประวัติแบบ Graph
git log --oneline --graph --all
🌿
ทำไมต้องใช้ Branch?
Branch ช่วยให้คุณพัฒนา feature ใหม่โดยไม่กระทบ code หลัก ใน DevOps เป็นเรื่องปกติที่จะทำงานบน branch แล้ว merge กลับเมื่อเสร็จ
✅ ผลที่ควรเห็น
  • กลับมาอยู่ branch main แล้ว
  • เนื้อหา README.md มีบรรทัดที่เพิ่มจาก feature branch
  • เห็น commit history ใน git log --oneline --graph

ชิ้นงานหลักของสัปดาห์นี้! Profile Repository คือ repository พิเศษที่ตั้งชื่อตรงกับ username ของคุณ — จะแสดงบนหน้า GitHub โดยอัตโนมัติ

สร้าง Profile Repository บน GitHub
▼
  1. ไปที่ github.com → คลิก "+" มุมบนขวา → "New repository"
  2. ตั้งชื่อ Repository ให้ ตรงกับ Username ของคุณพอดี (เช่น ถ้า username คือ somchai-devops ให้ตั้งชื่อว่า somchai-devops)
  3. ติ๊ก "Public" และ "Add a README file" → คลิก "Create repository"
  4. Clone ลงมาที่เครื่อง:
PowerShell — แทนที่ YOUR-USERNAME ด้วย username จริง
cd C:\devops-labs
git clone git@github.com:YOUR-USERNAME/YOUR-USERNAME.git
cd YOUR-USERNAME
code .
✅ ผลที่ควรเห็น
  • VS Code เปิดโฟลเดอร์ profile repository
  • มีไฟล์ README.md อยู่

Commit และ Push ≥ 5 ครั้ง
ชิ้นงานหลัก ▼
⚠️
สำคัญมาก: ต้องทำ commit อย่างน้อย 5 ครั้ง — ทำทีละนิดแล้ว commit ห้ามทำทีเดียวครั้งเดียว!
VS Code Terminal — ทำตามลำดับ ทีละ commit
# commit ครั้งที่ 1: โครงสร้าง README
git add README.md
git commit -m "docs: add profile README structure"

# แก้ส่วน "เกี่ยวกับฉัน" ให้ครบก่อน แล้ว commit ครั้งที่ 2
git add README.md
git commit -m "docs: add about me section"

# เพิ่มไฟล์ .gitignore
echo "node_modules/" > .gitignore
echo ".env" >> .gitignore
# commit ครั้งที่ 3
git add .gitignore
git commit -m "chore: add gitignore"

# แก้ checklist "สิ่งที่จะสร้าง" ให้ตรงกับความตั้งใจ
# commit ครั้งที่ 4
git add README.md
git commit -m "docs: update learning goals checklist"

# ใส่ username จริงในบรรทัด GitHub Stats
# commit ครั้งที่ 5
git add README.md
git commit -m "docs: add github stats badge"

# Push ทั้งหมดขึ้น GitHub
git push origin main
✅ ผลที่ควรเห็น
  • ไปที่ github.com/YOUR-USERNAME เห็น README แสดงบนหน้าโปรไฟล์
  • กด Commits ใน Repository เห็น ≥ 5 commits
  • ทุก commit message สื่อความหมาย ไม่ใช่แค่ "update" หรือ "fix"

Git Command Cheat Sheet

git init
เริ่ม repository ใหม่ในโฟลเดอร์ปัจจุบัน
git status
ดูสถานะไฟล์ (modified / staged / untracked)
git add <file>
เพิ่มไฟล์นั้นเข้า staging area
git add .
เพิ่มทุกไฟล์ที่เปลี่ยนแปลงเข้า staging
git commit -m "msg"
บันทึก snapshot พร้อม message
git log --oneline
ดูประวัติ commit แบบย่อ
git push origin main
ส่งขึ้น GitHub
git pull origin main
ดึงการเปลี่ยนแปลงจาก GitHub
git clone <url>
ดาวน์โหลด repository จาก GitHub
git checkout -b <name>
สร้าง branch ใหม่และเปลี่ยนไปใช้
git checkout main
กลับไป branch main
git merge <branch>
รวม branch เข้า branch ปัจจุบัน
git diff
ดูการเปลี่ยนแปลงที่ยังไม่ได้ stage
